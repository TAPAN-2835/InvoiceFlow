# InvoiceFlow AI (Invoxa)

A modern, high-performance invoicing and dashboard web application built with the latest front-end technologies. InvoiceFlow AI provides a seamless user experience with a beautiful UI, smooth animations, and robust routing.

## 🚀 Tech Stack

- **Framework**: [React 19](https://react.dev/) + [Vite](https://vitejs.dev/)
- **Routing & State**: [TanStack Router](https://tanstack.com/router/latest) & [TanStack Query](https://tanstack.com/query/latest)
- **Styling**: [Tailwind CSS v4](https://tailwindcss.com/)
- **UI Components**: [Radix UI](https://www.radix-ui.com/)
- **Animations**: [Framer Motion](https://www.framer.com/motion/)
- **Forms & Validation**: [React Hook Form](https://react-hook-form.com/) + [Zod](https://zod.dev/)
- **Language**: [TypeScript](https://www.typescriptlang.org/)
- **Icons**: [Lucide React](https://lucide.dev/)

## 🛠️ Features & Working

- **Seamless Routing**: Utilizing TanStack Router for type-safe, fast, and optimized client-side routing.
- **Beautiful UI**: Highly polished, modern aesthetics using Tailwind CSS and Radix UI primitives.
- **Form Handling**: Efficient, robust form validation utilizing React Hook Form and Zod schemas.
- **Data Fetching**: Optimistic updates and caching managed gracefully with TanStack Query.
- **Animations**: Fluid, delightful micro-interactions powered by Framer Motion.
- **Fully Responsive**: Designed to look and function perfectly across all device sizes.

## 📦 Getting Started

### Prerequisites

Ensure you have [Node.js](https://nodejs.org/) installed on your machine.

### Installation

1. Clone the repository and navigate to the project directory:
   ```bash
   cd invoiceflow-ai-main
   ```

2. Install dependencies:
   ```bash
   npm install
   # or
   bun install
   ```

3. Start the development server:
   ```bash
   npm run dev
   # or
   bun run dev
   ```

4. Open your browser and navigate to `http://localhost:5173`.

## 🏗️ Build for Production

To build the project for production, run:

```bash
npm run build
```

This will generate an optimized build in the `dist` directory.
